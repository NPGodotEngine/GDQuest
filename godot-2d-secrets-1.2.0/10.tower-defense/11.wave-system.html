Enemy Waves

In tower defense games, enemies usually come in groups we call waves. Levels have multiple waves and the player completes the level if they survive them all.

In this lesson, we’ll design a wave system that manages multiple enemies. It will manage the path of the enemies and when they spawn.

We’ll see how to spawn enemies in a predetermined order using the node tree and set a path for the enemies to walk dynamically at runtime.

Our enemies need a path and a wave

In the previous lesson, we created enemies that can follow a path. Our wave node will be that path, and a parent node will assign a curve based on available paths in the level.

Create a GDScript script directly here because we only need to create a Path2D node and attach the script to create a wave.

We add a list of enemies as children of that wave node and the script will make them spawn and move sequentially.

Create a new script file named Wave.gd in the Levels/Waves directory.

We start by defining some signals and properties.

class_name Wave
extends Path2D

## Emitted when calling the start function. It gives us an opportunity to
## display some text or an animation to announce the wave to the player.
signal starting
## Emitted when all enemies left the tree, that is, they were either killed by
## the player or damaged their base.
signal finished

## Delay in seconds between the spawn of two enemies.
## The export hint limits the slider's range between [0.1, 5.0] in the
## inspector.
## The third value, 0.05, is the step size to snap to.
export (float, 0.1, 5.0, 0.05) var enemy_time_interval := 0.5

Playing the wave

We add a start() function to allow another node to trigger a new wave. This could happen based on a timer or when the player clicks a button. It reinitializes enemy positions and makes them move sequentially.

We also connect to each enemy’s tree_exited signal to know when they died or damaged the player’s base.

func start() -> void:
    emit_signal("starting")
    _setup_enemies()
    _move_enemies()


# Resets the enemy's position on the path and connects to their `tree_exited`
# signal.
func _setup_enemies() -> void:
    for enemy in get_children():
        enemy.unit_offset = 0.0
        enemy.connect("tree_exited", self, "_on_Enemy_tree_exited")


# Moves the enemies sequentially, creating a timer of duration
# `enemy_time_interval` seconds to create a delay between them.
func _move_enemies() -> void:
    for enemy in get_children():
        yield(get_tree().create_timer(enemy_time_interval), "timeout")
        enemy.move()

Assigning a path

We provide a function to update the path the enemies will walk. The curve property below comes from the Path2D class, which we extend.

We design all the available paths on the level which will use pathfinding to find a way to reach the player base.

This allows us to reuse waves that move through different paths.

func set_movement_path(movement_path: PoolVector2Array) -> void:
    curve.clear_points()
    for point in movement_path:
        curve.add_point(point)

Completing the wave

Since our Wave node will only have enemies as its children, we use the child count to check if enemies are still alive. If there is none present, it means they’ve all died so we can move to the building phase.

func is_wave_finished() -> bool:
    return get_child_count() < 1


# When all enemies died, we end the wave and free the node.
func _on_Enemy_tree_exited() -> void:
    if is_wave_finished():
        emit_signal("finished")
        queue_free()

Godot’s node system keeps track of enemies for us and saves us time.

Creating and testing our first wave

We can already test a wave even without the rest of the game.

To do so, create a new scene with a Path2D node as the root. Add points to the path so the enemies can move along in the same way you did at the end of the previous lesson.

Attach the Wave.gd script to the node. You can tweak the time interval between enemy spawns in the Inspector.

Add several instances of the Enemy scene as children of the Path2D node.

To trigger the wave, you need to call start() from the wave script’s _ready() function. Be sure to remove this code after testing the scene.

func _ready() -> void:
    start()

And that is all you need. Save and run the scene to see the enemies appear and move one after the other along the path.
