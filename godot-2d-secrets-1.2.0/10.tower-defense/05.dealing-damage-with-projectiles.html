Dealing Damage with Projectiles

Bullets are the core of our tower defense combat. Towers shoot them to destroy enemies and protect our base.

Bullets need a position and rotation so we can start our scene with a Node2D as root. Rename it as Bullet and save it as res://Objects/Weapon2D/Bullets/Bullet.tscn.

Add a Sprite using the same process we did in the Weapon2D and select the region that represents a bullet.

You can also use the following values in the Sprite > Region > Rect to achieve the same result.

Our bullet will resemble a missile; it’s a projectile with a thrust attached. Attach another Sprite as a child of the first one and repeat the process for the thrust fire.

Move the second sprite a bit to the left so that the fire is in the right place. Check Visibility > Show Behind Parent to make sure it renders behind the first sprite.

A Bullet’s main responsibility is to carry a HitBoxArea2D to a target position. Add an instance of HitBoxArea2D and set the Team to the player’s team. As for the damage, let’s set it to 3.

Add a CollisionShape2D as child of the HitBoxArea2D. For its Shape property, we use a Capsule2D to mirror the shape of the sprite. Rotate it by 90 degrees and align it to the Sprite’s texture.

A Tween moves the Bullet. Add a Tween as a child of the Bullet scene and change the Tween > Playback Process Mode to Physics. This ensures the interpolation syncs with the physics process and the collisions from HitBoxArea2D happen at the right time.

We use an AnimationPlayer to cause the bullet to explode when it completes its trajectory.

Each bullet can explode in different ways. For instance, a bomb may have a large explosion, while a common bullet may have a smaller one. You can find the SmallExplosion.tscn file in the res://VFX/Explosion/ folder. Add an instance of it as child of the Bullet so we can create an animation with it.

This animation will be short to match the smaller projectile. Create the a new animation called Explode and set the time to 0.1 seconds.

After 0.1 seconds, toggle the emitting property off to release a short burst of particles at the start of the explosion.

We use the Call Method Track to dispose of the Bullet after the effect finishes. Call the queue_free() method on the Bullet at the end of the animation.

Attach a new GDScript to the Bullet and let’s make it fly! You can save the script together with the scene at res://Objects/Bullets/Bullet.gd.

It should fly at a given speed, so let’s export a speed variable. This allows us to set different flying speeds for bullets through the Inspector. Create a _tween variable to store a reference to the Tween node so we can use it later in the code.

class_name Bullet
extends Node2D

export var speed := 500.0

onready var _tween := $Tween
onready var _anim_player := $AnimationPlayer

The first thing we need to do is to make the Bullet a top-level node. This will prevent its parent from affecting its transform. We don’t want the rotation of the Weapon2D to interfere with the Bullet’s movement for example.

func _init() -> void:
    set_as_toplevel(true)

Let’s create the fly_to() method. As we saw before, the Weapon2D sends the target_global_position parameter when its found a target.

This is where we work out the path of the projectile. We use the distance_to() built-in method to work out the duration of the tween animation.

func fly_to(target_global_position: Vector2) -> void:

    # Calculate the duration of the tweening based on the distance to the target
    # position
    var distance := global_position.distance_to(target_global_position)
    var duration := distance / speed

    _tween.interpolate_property(
        self, "global_position", global_position, target_global_position, duration
    )
    _tween.start()

The Bullet should always point to its target’s position so we use the look_at method again.

We remove the Bullet from the scene when it’s finished moving to prevent it from hitting anything outside its range.

The yield() keyword waits for the _tween to complete its interpolations. Then we can play the Explode animation on the _anim_player. The explode animation calls the queue_free() method.

    # Rotates the bullet towards the target position
    look_at(target_global_position)

    # Wait for the tween to complete then call the explode animation that
    # removes the bullet from the scene
    yield(_tween, "tween_all_completed")
    _anim_player.play("Explode")

class_name Bullet
extends Node2D

export var speed := 500.0

onready var _tween := $Tween
onready var _anim_player := $AnimationPlayer


func _init() -> void:
    set_as_toplevel(true)


func fly_to(target_global_position: Vector2) -> void:
    var distance := global_position.distance_to(target_global_position)
    var duration := distance / speed

    _tween.interpolate_property(
        self, "global_position", global_position, target_global_position, duration
    )
    _tween.start()

    look_at(target_global_position)
    yield(_tween, "tween_all_completed")
